** hr_analytics_sql**

** Project Overview**

In this project, I analyzed an HR employee dataset using **SQL queries**, while the data itself was stored in **Excel**. The main purpose of this project is to understand employee-related patterns such as **attrition trends, salary distribution, department structure, job roles, and employee satisfaction**.

The project shows how raw HR data can be explored and analyzed using SQL to extract useful information that can help understand workforce behavior.


**# Data Source**

The dataset used in this project is stored in **Excel format**. It contains employee information from different departments in a company.

Some of the important columns in the dataset include:

* **EmpID** – unique ID for each employee
* **Age / AgeGroup** – employee age information
* **Department** – department where the employee works
* **JobRole** – employee job position
* **MonthlyIncome** – employee monthly salary
* **Attrition** – indicates whether an employee left the company or not
* **JobSatisfaction** – employee satisfaction with their job
* **EnvironmentSatisfaction** – satisfaction with the work environment
* **OverTime** – whether the employee works overtime
* **DistanceFromHome** – distance between home and workplace
* **NumCompaniesWorked** – number of companies the employee worked at previously


**# Project Workflow**

### 1. Data Collection

The HR employee data is stored in an **Excel file**, which acts as the main dataset for this project.

### 2. Data Import

The Excel dataset was imported into a **SQL database**, and a table called **hr_analytics** was created to store the data.

### 3. Data Exploration

Basic SQL queries were written to explore the dataset and understand the structure of the data.

### 4. Data Analysis

Multiple SQL queries were used to analyze the dataset and answer important HR-related questions such as employee distribution, attrition trends, and salary analysis.

### 5. Insight Generation

The results from SQL queries helped identify patterns in employee behavior and organizational structure.

**# SQL Concepts Used**

While working on this project, several SQL concepts were practiced:

* **SELECT** statements to retrieve data
* **WHERE** clauses to filter specific records
* **GROUP BY** to summarize data by categories such as department or job role
* **Aggregate functions** like `COUNT()`, `AVG()`, `MAX()`, and `SUM()`
* **CASE statements** for conditional analysis
* **Subqueries** for more advanced comparisons
* **ORDER BY** to sort and identify top or bottom results
